## Patterns Applied

### Worker-Style Singleton Pattern
```go
var (
    instance *Type
    once     sync.Once
)

func InitType(args) *Type {
    once.Do(func() {
        instance = &Type{...}
    })
    return instance
}

func GetType() *Type {
    return instance
}
```

### Logging Prefix Pattern
All server logs now use "server:" prefix for consistency with worker pattern
Examples:
- `log.Printf("server: connecting to database...")`
- `log.Printf("server: tree loaded: %d dirs, %d files", ...)`

## Remaining Design Differences (Acceptable)

### Server package remains request/response oriented:
- Handlers process HTTP requests (short-lived)
- WebSocket connections managed per-worker
- Database is singleton with connection pool

### Worker package is event/loop oriented:
- Components run forever in goroutines
- Explicit lifecycle management with Run/Close
- Reconnection and backoff handling
