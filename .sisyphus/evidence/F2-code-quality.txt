Recommendations

1) Align file ID typing end-to-end
- Change ItemStatus.FileID from int32 to int64 in internal/server/db.go and corresponding usage sites.
- This matches BIGINT and prevents overflow edge cases.

2) Defensively copy sha1 in ApplyReport
- Replace assignment with copied bytes (same approach already used in hydrate path):
  - st.fileHashes[fileID][workerID] = append([]byte(nil), sha1...)
- Adds safety against caller-side buffer reuse/mutation.

3) Reduce lock hold time during hydration (if startup latency/contension matters)
- Scan DB into temporary structures first, then acquire lock and apply in-memory.
- Keeps correctness while reducing blocking window.

4) Add one targeted unit test
- New test: mutate sha1 input after ApplyReport and assert stored hash/conflict state remains unchanged.

5) Address repo-wide formatting drift
- Run gofmt on cmd/verifier/main.go and cmd/worker/main.go to keep CI/style hygiene clean.
